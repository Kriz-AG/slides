HTTP Request Headers

  1. Content-Length: number-of-bytes
  2. Referer: referer-URL
  3. User-Agent: browser-type
  4. Accept-Encoding: encoding-method-1, encoding-method-2, ...
  5. Cookie: cookie-name-1=cookie-value-1, cookie-name-2=cookie-value-2, ...

HTTP Response Headers

  1. Cache-Control: max-age=3600, public
  2. Content-Encoding:gzip
  3. Content-Type:text/html; charset: utf-8
  4. Content-Length: 10000
  5. Connection: Keep-Alive

REST: Idempotent and safe methods

An idempotent HTTP method is a HTTP method that can be called many times without different outcomes

Safe methods are methods that can be cached, prefetched without any repercussions to the resource.
